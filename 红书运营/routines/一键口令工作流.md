# 口令「今天发什么」— 一键日更工作流（设计稿）

> 目标：你说一句 **「今天发什么」**，自动跑完 **选题（书或影+BGM）→ 文案初稿 → 设计说明/封面指令**，产出落在 `日记/YYYY-MM-DD/`，**发布**默认仍由你在小红书 App 内完成或二次确认后粘贴。

## 1. 触发方式（三选一，按你投入从低到高）

| 方式 | 做法 | 适合 |
|------|------|------|
| **A. 纯手动** | 每天开终端 `cd` 到本仓库，对 Claude Code 发：`今天发什么` | 立刻可用 |
| **B. Cursor / Claude 里自定义 Slash 或 Skill** | 将「今天发什么」绑定为一条 command，内嵌与本文件相同的系统提示 + 要读的文件列表 | 少打字 |
| **C. 定时（选题专用）** | 见 **`routines/每日选题自动化.md`** + `routines/daily_run_topic_april.sh` + `routines/launchd/*.example.plist`：每天固定时刻只生成 **`选题建议.md`** | 日更选题无人值守 |
| **C' 定时（全链路口令）** | 自行复制脚本的 `PROMPT` 为「今天发什么」同义长提示（见 §5 示例）— **API/时长更高**，建议人审 | 高自动、高风险 |

## 2. 一句话口令时，Agent 必做顺序

1. **读**：`aboutme.md`、`routines/网络采风规则.md`、**当日**`角落素材/YYYY-MM-DD-raw.md`（无则当「无舵向，自由采风」）  
2. **建目录**：`日记/YYYY-MM-DD/`（日期 = 今天，中国时区按你本机）  
3. **选题 April**：写 `选题建议.md`，**3 套** 方案；若允许 MCP，只在 **`网络采风白名单.md`** 内 Fetch；**书或影+BGM+正版渠道+口碑依据** 见 `aboutme` 1.2  
4. **自动裁定（无人拍板时）**：`aboutme` §1.2 已约定 **一律采用《选题建议》中的方案 1** 进入后续步骤。若你日后在 `aboutme` 中改为「须人工确认方案」，再在本步改回为 **只生成到 `选题建议.md` 并停**，文尾提示「请回复：用方案 1/2/3」。  
5. **文案 April**：`文案初稿.md`（用 `角落栏目模板.md`）；**默认**基于 **方案 1**（与上条一致）  
6. **设计 April**：`设计稿.md` + `assets/generated/` 的**生成说明**（如接生图 API 则出图，否则只给**可执行的设计 brief** 供你 Canva/稿定）  
7. **发布 April**（可选一键启用）：`发布日志.md` 的**草稿** + 检查清单，**不自动**调用发帖 API  

## 3. `aboutme` 与「选方案」状态（与 CLAUDE 对齐）

- **当前约定**：`aboutme` **§1.2** 已写 **无人拍板 → 采用方案 1**；与口令「**今天发什么**」一键串到 **文案 + 设计** 不冲突。  
- **若将来改回人工选案**：在 `aboutme` §1.2 删或改「方案 1」句，并写上「**每次须回复用方案 1/2/3**」；同步把本节第 2 步第 4 点改回半自动版。

## 4. 与「每天不用动手」的边界

- **能自动化**：读规则、建目录、写 MD、按白名单网采、生图**指令或 API 出图**、飞书/钉钉 文本通知。  
- **难全自动且建议保留人审**：**盗版/错曲名/错平台在架**、**封面侵权**、**敏感词**、**你账号当日真实心情**与热点——至少 **5 分钟扫一眼** 再发。  
- **发帖**：`CLAUDE.md` 默认不自动发；若未来接官方/合规工具，**另开开关**，勿与「生成稿」混为一谈。

## 5. 非交互一键命令示例（B/C 用）

在项目根、且已 `export` 好账号相关环境时（示例，以你本机 claude 为准）：

```bash
cd "/Users/zhaoziqi/Documents/红书运营" && claude -p "今天发什么：请按 routines/一键口令工作流.md 与 aboutme 执行到文案初稿与设计稿，日期用本机今天；不要发帖。"
```

（是否支持 `-p` 长任务、以及是否要 `--allowedTools`，以当前 Claude Code CLI 文档为准。）

## 6. 安全

口令触发等于 **高权限自动写仓库**；**不要**在公开仓库的 CI 里用真实 Cookie；**`.secrets/`** 仅本机。
