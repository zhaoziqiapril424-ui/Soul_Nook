# 网络采风规则 — 选题 / 文案 April（书影音清单）

## 目标

允许 Agent **自行从公开网络** 搜集 **书单 / 影单 / 歌单** 素材，并在产出中写明 **可在哪些正版或官方渠道** 阅读、观看、收听（**须标注地区/会员等前提**）。

## 选题偏好（与 `aboutme` §1.2 对齐）

- **优先挖掘**：独立/作者向、近年的、讨论区有「争议或细读」而非只有金句搬运的书影； festival 展映、文学奖短名单、纪录片、非英语片等可作为方向。  
- **主动回避**：国民级现象级、自媒体写滥的「必提经典」、纯百科式热点；若采风结果多为常见名，**扩大检索词或换源**再出 3 套方案。  
- **真知灼见**：每条方案除元数据外，须有一句 **可辩论的观点钩子**（为何值得在今天写、和读者什么关系），**不得** 只有剧情简介。

## 允许的技术路径

- 使用已配置的 **MCP Fetch / HTTP**（如 Smithery Fetch）仅请求 **HTTPS 公开页面**。  
- 优先读 **`aboutme.md` 中「观看与收听习惯」** 所列地区与平台；未写明的须 **显式假设**（例如「以下以中国大陆常用平台为例」或「以北美订阅制平台为例」）并提醒读者自行核对。

## 禁止

1. **登录**任何平台账号、携带 Cookie、破解付费墙、绕过 DRM。  
2. 推荐或收录 **盗版/磁力/全集在线观看** 等非法观看渠道。  
3. 对 **未公开的付费站内片单** 做批量抓取或逆向。  
4. 把 API Key、Webhook、个人 token 写入仓库或 Markdown。  
5. 高频率、大并发请求同一站点（避免被视为攻击或爬虫滥用）。

## 信息来源与核验

- 每条书/影/歌建议 **至少标注一类来源**：官方页面、维基百科作品页、豆瓣/Goodreads/IMDb 等 **公开元数据页**（择与你地区可信者）。  
- **上映/上架/版权** 信息易变：正文或选题中写 **「以发布日检索为准，欢迎读者指正」**。  
- 影剧 **是否在架**（某区 Netflix / 爱奇艺等）不确定时，写 **「可能轮换下架，请在 App 内搜索全名核对」**。

## 选题建议.md 中每条方案须包含

1. 作品名（中/英可并列）与一句 **无剧透** 情绪切入点。  
2. **类型**：书 / 影 / 歌。  
3. **观看或阅读或收听**（按类型勾选）：列出 **正版渠道名称**（如：微信读书、豆瓣阅读、纸质书；爱奇艺、B 站、Disney+、Apple Music、网易云等），并注明 **地区 / 会员 / 租赁** 等前提。  
4. **信息来源**：可点击的公开链接或页面标题 + 站点名。  
5. 与当日 `角落素材`（若有）的 **对齐说明**：采风结果如何回应用户当日心情句。

## 白名单

具体允许访问的 **域名或 URL 前缀** 由维护者在 **`网络采风白名单.md`** 中列出；Agent **不得**访问该文件未列出的域名（除非你在对话中临时明确追加本次会话允许列表）。
