# 每日自动跑「选题 April」（本机 + launchd）

目标：在固定时刻（如每天 **7:00** 北京时间）由 **Claude Code CLI** 非交互生成当日的 `日记/YYYY-MM-DD/选题建议.md`。**不**写文案/设计/发帖。

## 前置条件

1. 本机已能跑：`claude -p`（`claude --help` 有 `-p`/`--print`）。
2. 已设 **`ANTHROPIC_BASE_URL` / `ANTHROPIC_API_KEY`**（智谱时与终端 `claude --bare` 相同）。
3. 电脑在定时时刻 **需开机**（若休眠未唤醒，launchd 可能顺延或错过后补跑，以系统表现为准）。

## 1. 环境变量（推荐单独文件，供定时任务用）

`launchd` **不会**读你的 `~/.zshrc`，请把与智谱/Anthropic 相关的 export 写进家目录、**不提交** 的文件，例如 `~/.secrets/hongshu_claude.env`（`chmod 600`）：

```bash
export ANTHROPIC_BASE_URL="https://open.bigmodel.cn/api/anthropic"
export ANTHROPIC_API_KEY="（你的 Key，勿出现在仓库）"
```

脚本 `routines/daily_run_topic_april.sh` 若发现该文件会自动 `source`。**勿**把 Key 写进 plist 或本仓库任何 Markdown。

## 2. 手动试跑

```bash
export TZ=Asia/Shanghai
"/Users/zhaoziqi/Documents/红书运营/routines/daily_run_topic_april.sh"
```

成功后检查 `日记/今日日期/选题建议.md` 与 `routines/logs/daily-topic-*.log`。

若权限弹窗/无法写文件，可临时设置更保守的权限模式（在跑脚本**前** `export`）：

```bash
export HONGSHU_CLAUDE_PERMISSION_MODE=acceptEdits
# 或（仅本机、仓库可信时）默认脚本已用 bypassPermissions
```

## 3. 安装 launchd（macOS）

1. 复制并编辑 `routines/launchd/com.hongshu.daily-topic-april.example.plist`：把里面路径、**Hour/Minute** 改成你的习惯（示例为 7:00）。  
2. 安装：

```bash
cp routines/launchd/com.hongshu.daily-topic-april.example.plist ~/Library/LaunchAgents/com.hongshu.daily-topic-april.plist
launchctl load ~/Library/LaunchAgents/com.hongshu.daily-topic-april.plist
```

3. 查看是否在列表：`launchctl list | grep hongshu`  
4. 卸载：

```bash
launchctl unload ~/Library/LaunchAgents/com.hongshu.daily-topic-april.plist
```

`routines/logs/` 已加入 `.gitignore`，**不要**把带隐私的整份 log 推送到公开库。

## 4. 与「今天发什么」全链路的关系

- **本脚本**只到 **选题**；若你想要定时跑「选题 + 文案 + 设计」，可另复制脚本、把 `PROMPT` 改成全链路（或把自然语言设成与 `routines/一键口令工作流.md` 中一致）——**成本与错链风险**更高，建议仍保留人工扫一眼。  
- 你当前 `aboutme` 若约定「无人拍板 → 方案 1」，**选题之后**的文案/设计在交互会话里用 Cursor 做即可。

## 5. 风险提醒

定时任务 = 自动**读/写**本仓库。仅在本机、仓库路径可信、Key 不泄露的前提下使用。若 Key 曾出现在聊天或历史命令里，请先在控制台**轮换**。
