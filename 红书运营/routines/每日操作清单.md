# 每日操作清单 — 在哪儿、做什么

> 本仓库根目录：**`红书运营/`**（下文「仓库根」均指此处）。  
> 绝对路径示例：`/Users/zhaoziqi/Documents/红书运营`（若你本机不同，请自行替换）。

---

## 一、一次性准备（做过可跳过）

### 1. 配 API Key（给定时选题 + `daily_run_topic_april.sh` 用）

| 在哪儿 | 做什么 |
|--------|--------|
| 用编辑器打开 **`~/.secrets/hongshu_claude.env`**（家目录下隐藏文件夹 `.secrets`） | 填写 **`ANTHROPIC_API_KEY`**；**`ANTHROPIC_BASE_URL`** 按你使用的服务商保持与 [`每日选题自动化.md`](每日选题自动化.md) 一致。保存。 |
| Finder | 若看不到 `.secrets`：按 **`Cmd + Shift + .`** 显示隐藏文件。 |

**注意**：不要把 Key 写进本仓库任意 `.md`、脚本或 Git 跟踪文件。详见仓库内 **`.secrets/README.md`**（若存在）。

### 2. 让终端认识 `hongshu` / `hongshu-today` / `ccxh`

| 在哪儿 | 做什么 |
|--------|--------|
| 任意终端 | 执行 **`source ~/.zshrc`**；或**关掉终端再开**一次。 |

（上述别名与函数应在你的 **`~/.zshrc`** 中已配置；若缺失，见 [`CLAUDE.md`](../CLAUDE.md) 与 [`每日选题自动化.md`](每日选题自动化.md)。）

### 3. launchd（每天早上只生成选题）

| 在哪儿 | 做什么 |
|--------|--------|
| **`~/Library/LaunchAgents/com.hongshu.daily-topic-april.plist`** | 若要改执行时间：编辑 **`Hour`**、**`Minute`**，保存后执行：`launchctl unload ~/Library/LaunchAgents/com.hongshu.daily-topic-april.plist`，再 `launchctl load …`。 |
| 本机 | 到点需**开机**；休眠未唤醒时任务可能不跑或延后（以系统为准）。 |

安装与卸载步骤全文：**[`每日选题自动化.md`](每日选题自动化.md)**。

## 二、你每天可以怎么用（选一种或组合）

### 方式 A：终端 + 交互式 Claude（推荐日常）

| 步骤 | 在哪儿 | 做什么 |
|------|--------|--------|
| 1 | 终端 | 输入 **`ccxh`** → 会 `cd` 到仓库根并启动 `claude --bare`（带 `routines/agents.json`）。 |
| 2 | 同一终端里的 Claude | 按 **[`CLAUDE.md`](../CLAUDE.md)** 触发语：例如 **「开始今天的工作」** → 等 **`日记/YYYY-MM-DD/选题建议.md`**；再 **「用方案 N」** → 文案 → **「文案定稿，做图」** → **「终稿，走发布检查」**。 |
| 3（可选） | 仓库 **`角落素材/`** | 若当天有舵向，先写好 **`YYYY-MM-DD-raw.md`**，再说「开始今天的工作」。 |

### 方式 B：只进目录，自己再开 claude

| 步骤 | 在哪儿 | 做什么 |
|------|--------|--------|
| 1 | 终端 | **`hongshu`** → 只 `cd` 到仓库根。 |
| 2 | 终端 | 自行 **`claude`** / **`claude --bare`**（需当前环境已配置 API 相关变量）。 |

### 方式 C：一键「今天发什么」（非交互，到文案 + 设计）

| 步骤 | 在哪儿 | 做什么 |
|------|--------|--------|
| 1 | 终端 | 确保 Key 已在 **`~/.secrets/hongshu_claude.env`** 或当前 shell 中可用。 |
| 2 | 终端 | **`hongshu-today`** → 按 **[`一键口令工作流.md`](一键口令工作流.md)** 执行（默认**不发帖**）。 |
| 3 | **`日记/YYYY-MM-DD/`** | 检查 **`选题建议.md`、`文案初稿.md`、`设计稿.md`**，人工修订后再发小红书。 |

### 方式 D：定时只出选题（launchd）

| 步骤 | 在哪儿 | 做什么 |
|------|--------|--------|
| 1 | 到点后（自动） | 生成 **`日记/YYYY-MM-DD/选题建议.md`**（不写文案/设计/发帖）。 |
| 2 | **`routines/logs/`** | 查看 **`daily-topic-日期.log`**、**`launchd-stdout.log`**、**`launchd-stderr.log`** 排查错误。 |

## 三、用 Cursor 时（可选）

| 在哪儿 | 做什么 |
|--------|--------|
| Cursor | **File → Open Folder**，打开本仓库 **`红书运营`** 根目录。 |
| Cursor 内置终端 | 先 **`hongshu`** 或 **`cd` 到仓库根**，再执行命令。 |
| 规则来源 | **`.cursor/rules/心灵角落-总规则.mdc`**、**[`CLAUDE.md`](../CLAUDE.md)**、**[`aboutme.md`](../aboutme.md)**。 |

---

## 四、发帖前（人工）

| 在哪儿 | 做什么 |
|--------|--------|
| 小红书 App / 创作者后台 | 从 **`发布日志.md`** 或定稿后的 **`文案初稿.md`** 复制正文与标签。 |
| 仓库 | 对照 **[`发布检查清单.md`](发布检查清单.md)** 过一遍再发布。 |

---

## 五、最小每日顺序（速查）

1. **`~/.secrets/hongshu_claude.env`**：Key 已配置（Key 轮换时更新）。  
2. 终端 **`source ~/.zshrc`**（或新开终端）。  
3. **`ccxh`**，或 **`hongshu`** + 手动 `claude`，或 **`hongshu-today`**。  
4. **`日记/今天日期/`** 下改稿、定稿、按需更新 **`设计稿.md`** / 出图。  
5. **小红书** 内粘贴发布。
