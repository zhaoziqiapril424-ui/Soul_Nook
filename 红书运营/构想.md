# April 五人团队 — 小红书「心灵角落」账号运营构想（可落地版）

> **一句话**：以 **「心灵角落」** 为 **唯一主叙事**，用 **Claude Code + 五个 April（Subagent）** + **`aboutme.md` + 按日归档**，把小红书日更拆成 **可重复的「选题 → 文案 → 设计 → 发布检查 → 复盘」**；**选题 April 可在白名单 + MCP Fetch 下自主网络采风**，产出书/影/歌素材并 **写明正版或官方观看与阅读收听渠道（含地区/会员前提）**；旅行、美食、金句、偶发求职话题 **只做主叙事下的变体**；**禁止** 登录爬取、盗版链接与全自动发帖。

---

## 一、账号定位（已拍板 · 运营真源）

### 1.1 主叙事：心灵角落

- **帮谁**：觉得累、卷、对未来不安，仍想保留 **一小块能喘气、能被听懂** 的读者（也是过去的自己）。  
- **提供什么**：每天一个 **很小的入口**——优先用 **一本书 / 一部电影 / 一首歌** 带出一个 **可共鸣的情绪 + 一个开放式提问**；不灌鸡汤 KPI，不制造「你必须每天自愈」的 **假疗愈**。  
- **我是谁（写作人格）**：ENFJ、爱观察、爱碎碎念、爱交友；语气 **像朋友递水**，承认局限、邀请评论区一起补一句真实故事。别人记住你的点：**通人性、周到、不装全知**（写入 `aboutme.md` 后由全体 April 遵守）。

### 1.2 固定栏目（日更默认）

| 栏目名（可改） | 结构（选题/文案 April 默认遵守） | 说明 |
|----------------|----------------------------------|------|
| **「角落一分钟」**（示例名） | **1 个作品**（书/影/歌三选一）+ **1 段碎碎念** + **1 个提问** + 可选 **一句金句**（注明出处或标明为原创概括） | 控制篇幅，避免日更变薄；歌词/台词注意 **引用长度与版权习惯**。 |

**周更 1～2 次（建议写进 `aboutme.md`）**：同一作品或同一情绪的 **加长篇**（多图大纲 + 分段小标题），冲收藏与深度关注。

### 1.3 副线（允许，但必须挂回主叙事）

旅行、美食、情绪金句、生活碎片 **可以发**，但必须能回答一句话：

> **「它和今天这个『角落』里的情绪，有什么关系？」**

没有关系则 **不发**，或降级为 **极低频纯生活**（建议仍少发，防算法标签漂移）。

### 1.4 求职 / 行业内容（弱关联，不做第二主叙事）

- **不做**：求职陪跑、改简历套餐、大厂优绩路径贩卖、制造焦虑的对比叙事。  
- **可做**：仅在 **与当日情绪自然接壤** 时出现（例如「这首歌让我想起投简历那段日子」「这部电影里有一种不被 KPI 定义的人」），篇幅与频率 **由你在 `aboutme.md` 里设上限**（例如每周不超过 1 篇或仅出现在提问里）。

### 1.5 不做清单（硬）

1. 短期 **不做纯追星**、不做与主叙事无关的 **带货测评**。  
2. **不做优绩主义叙事**（不晒无信息量「我很卷」、不暗卷读者）。  
3. **不做假疗愈**（空喊加油、空洞口号、把读者当情绪垃圾桶无回馈）。  
4. **不做无法核验的「行业/求职断言」**；若涉及现实信息，采用 **现象 + 信息来源 + 欢迎从业者纠偏**，避免造谣与恐慌传播。  
5. **技术向**：**禁止** 未授权登录爬取、Cookie 入仓库、默认自动发帖、盗版与破解付费墙；**允许** 在 **`routines/网络采风白名单.md`** 与 **`routines/网络采风规则.md`** 约束下，经 **MCP Fetch** 访问 **公开 HTTPS** 页面以整理书单/影单/歌单与 **正版渠道说明**。

### 1.6 与设计原则（工程侧）

1. **可审计**：当日产出落在 `红书运营/日记/YYYY-MM-DD/`。  
2. **合规优先**：社区规范、敏感词、引流话术、书影音引用尺度。  
3. **密钥隔离**：飞书/钉钉、生图、API 仅环境变量或 `.secrets/`，不入库、不写进提示词。

### 1.7 书影音网络采风与「在哪里看 / 读 / 听」

- **业务目标**：Agent 可自行从网络补充 **书单 / 影单 / 歌单** 候选，并在 `选题建议.md` / `文案初稿.md` 中写明 **正版或官方渠道**（如微信读书、豆瓣、各大视频 App、Apple Music、网易云等），并标注 **地区、会员、租赁、是否可能下架轮换** 等前提。  
- **工程约束**：仅通过 **已配置的 MCP Fetch**（例如 Smithery：`https://fetch--smithery-ai.run.tools`）访问 **`网络采风白名单.md`** 中的域名；遵守 **`网络采风规则.md`**。  
- **免责声明**：版权与上架状态易变，正文须含 **「以读者本地 App 检索为准」** 类提示，避免绝对承诺「一定能看」。  
- **与 `角落素材/` 的关系**：`角落素材/YYYY-MM-DD-raw.md` 为 **可选舵向**（心情、指定作品、禁止话题）；若为空或仅一句话，选题 April **仍以采风补全候选**，但不得违反 `aboutme` 与不做清单。

## 二、五位 April（按「心灵角落」重做职责）

| 序号 | 名称 | 触发语（示例） | 在本账号下的核心产出 |
|------|------|----------------|----------------------|
| 1 | **选题 April** | 「早上好」「开始今天的工作」 | `选题建议.md`：3 套 **「角落一分钟」** 方案 + 飞书/钉钉卡片 |
| 2 | **文案 April** | 你 **选定今日方案** 后 | `文案初稿.md`：碎碎念正文 + **5 标题**（标注偏情绪共鸣 / 偏搜索）+ 标签与提问 |
| 3 | **设计 April** | 你 **确认文案** 后 | `设计稿.md` + `assets/generated/`：偏 **留白、柔和、大字+轻配图** 的封面思路 |
| 4 | **发布 April** | 你 **确认终稿** 后 | 发布检查 + `发布日志.md` + 内容库索引（**默认手动发帖**） |
| 5 | **复盘 April** | **次日/周末** + 有数据 | `数据复盘.md` + 仅追加 `aboutme.md`「账号迭代记录」 |

### 2.1 选题 April

- **必读**：`aboutme.md`；`routines/网络采风规则.md`；`routines/网络采风白名单.md`。  
- **输入**：`角落素材/YYYY-MM-DD-raw.md`（**可选**：心情句、指定作品、今日想避开的主题）；若文件缺失或极短，**不阻止**采风。  
- **网络采风**：使用 **MCP Fetch** 仅在白名单域名内检索公开页面，整理 **书单 / 影单 / 歌单** 候选；**每条候选必须**包含：**作品名、类型（书/影/歌）、一句情绪切入点、正版观看/阅读/收听渠道列表（含地区或会员前提）、信息来源 URL**。**禁止** 盗版站、登录态抓取、付费墙破解。  
- **流程**：输出 **3 套** 可拍板方案；每套除上述字段外，含：**封面气质一句话、给读者的提问、是否建议周更加长**；若用了采风，在方案末附 **「采风来源摘要」**（链接列表）。  
- **产出**：`日记/YYYY-MM-DD/选题建议.md` + 飞书/钉钉卡片。

### 2.2 文案 April

- **必读**：选定方案 + `aboutme.md`。  
- **流程**：写成 **「角落一分钟」** 体例；**5 个标题**：区分 **情绪/共鸣向** 与 **可检索向**（如作品名+情绪词）；**结尾固定留一个具体问题**；Emoji 与分段符合小红书阅读习惯；**禁止** 不做清单中的话术。  
- **观看渠道**：在正文 **固定增加一小节**（标题自拟，如「在哪里读 / 看 / 听」），与 `选题建议.md` 一致或略细化；**仅列正版或官方渠道**，写法与免责见 `网络采风规则.md`；**禁止** 附盗版链接。  
- **产出**：`文案初稿.md`。

### 2.3 设计 April

- **必读**：定稿文案 + `aboutme.md` 视觉规范（若暂无成体系 VI，则默认 **低饱和、留白、单主色**）。  
- **流程**：**1 主封面 + 0～2 张内页** 思路即可（心灵号不宜过重包装）；若有 **剧照/专辑封面** 元素，在 `设计稿.md` 注明 **建议使用原创字卡+弱引用**，避免侵权雷区。  
- **产出**：`设计稿.md` + 生成图说明；飞书预览（可选）。

### 2.4 发布 April

- **必读**：终稿、`routines/发布检查清单.md`。  
- **流程**：敏感词、引流、标签数量、图片清晰度检查；生成 **可复制到小红书发布页的清单**；写 `发布日志.md` 与内容库索引。  
- **默认**：不调用发帖 API。

### 2.5 复盘 April

- **必读**：`发布日志.md`、`内容库/`、`数据输入/`、`aboutme.md`。  
- **流程**：在心灵号维度上多看 **收藏、评论、私信关键词**（你粘贴进 `数据输入`）；提炼 **一条** 下周可执行洞察（例如：「提问型结尾比金句型标题评论更高」）。  
- **产出**：`数据复盘.md`；**仅追加** `aboutme.md` 中 **账号迭代记录**。

## 五、可行性评估（心灵号特有风险）

| 环节 | 可行性 | 说明 |
|------|--------|------|
| 文档链 + 飞书审批 | **高** | 与工具链一致。 |
| 日更「角落一分钟」质量 | **高** | 依赖 **模板 + 采风 + 可选角落素材舵向**。 |
| 公开页采风（白名单 + MCP Fetch） | **中高** | 需你维护 **白名单**；渠道信息 **易变**，须固定免责句式；**不接** 登录与盗版。 |
| 书影音引用与封面用图 | **中** | 长引用、剧照、专辑封面有 **版权与平台规则** 风险；优先 **原创字卡 + 短摘 + 来源说明**。 |
| 无 MCP 时的采风 | **低** | 无 Fetch 则选题 April **仅能用**已打开的页面摘要或 `角落素材` 全文；建议配置 **Fetch MCP**。 |
| 一键自动发帖 / 全自动拉数 | **中～低** | 同前版结论：默认手动 + `数据输入/` 粘贴。 |

## 九、MCP 与自动化（书影音采风）

| 优先级 | 工具 | 用途 |
|--------|------|------|
| **P0（强烈建议）** | **Fetch 类 MCP** | 选题 April 按 **`网络采风白名单.md`** 拉取公开 HTML，组装书单/影单/歌单与 **正版观看渠道** 说明。你可使用 Smithery 托管入口（示例）：`https://fetch--smithery-ai.run.tools` —— **具体接入方式以 Smithery / Claude Code 当前文档为准**，并在 MCP 配置中绑定为只读 HTTP。 |
