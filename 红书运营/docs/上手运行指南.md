# 上手运行指南 — 克隆本仓库后如何在本机跑起来

写给第一次接手本项目的人：**你需要打开对的文件夹、配置好自己的 API Key，再用 Cursor 或终端里的 Claude Code 驱动 Markdown 工作流。**

**更短的三段式清单**（迷茫期 / 配置期 / 运行期）：见 **[`协作者从clone到第一句话.md`](协作者从clone到第一句话.md)**。

默认**不会自动发帖**；配图可走即梦网页/App，不必接入生图 API。

**要上 GitHub / 给他人**：另见 **[`GitHub上传与运行指南.md`](GitHub上传与运行指南.md)**（推送检查清单、`git clone` 后最短路径）。

---

## 一、先确认：哪个文件夹才是「项目根」？

本仓库的内容对应你本地的 **`红书运营`** 文件夹：**上传到 GitHub 时，应把这些文件放在仓库根目录**（与 [`CLAUDE.md`](../CLAUDE.md) 所在层级一致），而不是再多套一层文件夹。

**推荐仓库布局**：把整个 **`红书运营`** 文件夹里的内容放在 Git 仓库的**根目录**（clone 下来第一层就看到 `CLAUDE.md`、`README.md`、`routines/`），不要再只用于本地分类的外层父目录里多套一层，以免找不到真正的项目根。

```text
你的仓库根（clone 后进入的这一层）
├── README.md
├── CLAUDE.md
├── aboutme.md
├── routines/
├── 日记/
└── …
```

**判断标准**：在 Finder 或编辑器里能看到 **`CLAUDE.md`** 的那一层，就是下文所说的「仓库根」或「项目根」。  
若你手上仍是旧的多层嵌套路径，请只对「里面有 `CLAUDE.md`」的那一层开 Cursor / 执行终端命令。

## 二、环境清单（一次性准备）

**不是 Node 项目**：仓库根**没有** `package.json`，也**不需要** `npm install`。依赖是编辑器 +（可选）CLI 与 Python。

| 需要 | 用途 |
|------|------|
| **Git** | `git clone` 仓库 |
| **[Cursor](https://cursor.com/)**（推荐）或 VS Code | 打开项目根；内置 Chat / Agent 读 `.cursor/rules/` |
| **[Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code/overview)**（终端命令 `claude`） | 与 [`routines/agents.json`](../routines/agents.json) 子 Agent 流程一致 |
| **Python 3**（可选） | 仅在使用 [`daily_run_topic_april.sh`](../routines/daily_run_topic_april.sh) / launchd 定时选题时需要 |
| **MCP Fetch**（可选） | 选题阶段「网络采风」：仅能访问 [`网络采风白名单.md`](../routines/网络采风白名单.md) 中的域名；细则见 [`网络采风规则.md`](../routines/网络采风规则.md) |

## 三、API Key（必做，且永远不要提交到 Git）

1. 在 [Anthropic Console](https://console.anthropic.com/)（或你所用的兼容服务商控制台）创建 **API Key**。  
2. 在本机家目录创建私密文件（路径固定，便于与脚本一致）：

   ```bash
   mkdir -p ~/.secrets
   chmod 700 ~/.secrets
   ```

3. 新建或编辑 **`~/.secrets/hongshu_claude.env`**（权限建议 `chmod 600`），例如：

   ```bash
   export ANTHROPIC_API_KEY="sk-ant-api03-……"
   ```

   若使用国内兼容网关，可能还需要 **`ANTHROPIC_BASE_URL`**，见 [`routines/每日选题自动化.md`](../routines/每日选题自动化.md) 与脚本注释。

4. **不要把 Key 写进** 仓库里的 Markdown、也不要 `git add` 任何含密钥的文件。说明见 [`.secrets/README.md`](../.secrets/README.md)。

## 四、方式 A：用 Cursor（不写终端命令也能干活）

1. **File → Open Folder**，选择 **含 `CLAUDE.md` 的项目根**（见上文第一节）。  
2. 在 Chat / Agent 里用自然语言即可；仓库规则来自 **`.cursor/rules/心灵角落-总规则.mdc`**、**`CLAUDE.md`**、**`aboutme.md`**。  
3. 若需在终端执行脚本，可在 Cursor 内置终端里先 `cd` 到同一项目根。

**说明**：Cursor 内模型的计费与 Anthropic 官网额度可能分开；若在 Cursor 里配置了自家的模型/API，行为可能与终端 `claude` 略有差异，但 Markdown 产出路径不变（仍以 **`日记/YYYY-MM-DD/`** 为准）。

---

## 五、方式 B：终端 + Claude Code CLI（与脚本、子 Agent 对齐）

每次新开终端建议：

```bash
cd "/你的路径/xinlingjiaoluo"   # clone 下来的仓库根（这一层须有 CLAUDE.md）

source ~/.secrets/hongshu_claude.env

claude --bare --agents "$(cat routines/agents.json)"
```

进入交互后，按 **[`CLAUDE.md`](../CLAUDE.md)** 里的触发语操作即可（例如「开始今天的工作」「用方案 2」「文案定稿，做图」等）。

**若 `--agents` 解析报错**：可参考 [`routines/daily_run_topic_april.sh`](../routines/daily_run_topic_april.sh) 里用 `python3` 把 `agents.json` 压成单行再传给 `claude` 的做法。

### 可选：缩短命令（自己配置别名）

仓库作者在 **[`routines/每日操作清单.md`](../routines/每日操作清单.md)** 里提到的 **`hongshu` / `ccxh` / `hongshu-today`** 需要在你在本机的 **`~/.zshrc`** 里自行定义（Git 仓库通常不包含别人的家目录配置）。新人可直接使用上面的 **`cd` + `source` + `claude`** 三行，效果等价。

## 六、产出在哪里？第一天可以做什么？

| 动作 | 典型产出 |
|------|-----------|
| 「开始今天的工作」 / 选题 April | `日记/YYYY-MM-DD/选题建议.md` |

## 七、合规与安全（请务必遵守）

- 禁止向仓库提交 **Cookie、Token、盗版链接、真实 API Key**。  
- 网络请求仅限 **[白名单](../routines/网络采风白名单.md)** 与 **[采风规则](../routines/网络采风规则.md)**。  
- 默认 **不** 调用小红书发帖 API；配图勿 1:1 复刻官方海报（见 **`aboutme.md`**）。
