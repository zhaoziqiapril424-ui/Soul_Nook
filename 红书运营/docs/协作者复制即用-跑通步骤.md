# 协作者复制即用：跑通步骤

发给第一次接手的人：**按顺序做即可**。更细的背景说明（A/B 接法、密钥原理、路径排查）见 **[`协作者从clone到第一句话.md`](协作者从clone到第一句话.md)**。

---

## 0. 项目根在哪

Clone 后若第一层只看到文件夹 **`红书运营`**，请先进入该文件夹。  
**项目根** = 这一层里能直接看到 **`CLAUDE.md`** 和 **`README.md`** 的目录。

之后：**终端命令**、**Cursor / Web IDE 打开文件夹**，都以这一层为准。

---

## 1.（可选）初始化常用目录

在项目根执行：

```bash
sh ./routines/bootstrap_after_clone.sh
```

看到类似 **`OK：目录已就绪`** 即可。  
若同时提示未找到 `~/.secrets/hongshu_claude.env`，在 Web 环境或未配本机密钥时**可忽略**（本脚本不读取密钥）。

---

## 2. 二选一：你怎么用工具

### 情况 A：用 Claude Code / Cursor 打开工程（浏览器里的在线版也算）

1. 把工作区根设为 **含 `CLAUDE.md` 的项目根**（见上一节）。  
2. 在对话里发送：**「开始今天的工作」**  
   （更多口令见同目录 **[`CLAUDE.md`](../CLAUDE.md)** 中的表格。）  
3. **预期结果**：**`日记/`** 下出现 **当天日期**（`YYYY-MM-DD`）文件夹，其中有 **`选题建议.md`**（具体以实际生成为准）。

**说明**：鉴权一般由**平台或客户端**处理，通常**不需要**在本机建 `~/.secrets`。若平台仍要求填写 API 或网关，按其说明配置；**不要把 Key 写进仓库、不要截图发公开场合**。

---

### 情况 B：本机终端使用 `claude`，且需国内/兼容网关（Anthropic 兼容接口）

1. 在本机执行：

```bash
mkdir -p ~/.secrets && chmod 700 ~/.secrets
```

2. 在 **项目根**（含 `CLAUDE.md`）执行：

```bash
cp .secrets/hongshu_claude.env.example ~/.secrets/hongshu_claude.env
chmod 600 ~/.secrets/hongshu_claude.env
```

3. 用编辑器打开 **`~/.secrets/hongshu_claude.env`**，至少填写（按模板内注释为准）：

   - **`export ANTHROPIC_API_KEY="你的 Key"`**  
   - **`export ANTHROPIC_BASE_URL="服务商提供的地址"`**（不使用网关则按模板注释处理）

   URL 与网关示例可参考 **[`routines/每日选题自动化.md`](../routines/每日选题自动化.md)**；**非同一服务商时以对方文档为准**。

4. **每次**新开终端后，先执行：

```bash
source ~/.secrets/hongshu_claude.env
```

再 **`cd`** 到 **项目根**，运行 **`claude`**，然后发送：**「开始今天的工作」**。

**不要**使用 `export $(cat ~/.secrets/... | xargs)` 等方式加载密钥（易出错且不安全）。

---

## 3. 仍失败时请回传

便于排查的两项信息：

1. **终端完整报错**（或截图）。  
2. 在项目根执行 **`pwd`** 的输出（确认是否与 `CLAUDE.md` 同级）。
